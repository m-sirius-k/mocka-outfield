# MoCKA Outfield Constitution

## 1. Purpose

The Outfield is the public cognitive surface of the MoCKA system.

It provides a controlled interface through which external agents,
researchers, and systems can observe and interact with MoCKA knowledge.

The Outfield contains only sanitized and publishable artifacts.

## 2. Boundary Principle

MoCKA is composed of two primary domains.

Infield  
Internal reasoning, experiments, operational memory, and system state.

Outfield  
Public knowledge, exported records, collaboration interfaces,
and externally visible system artifacts.

The Infield remains the authoritative internal reasoning layer.

## 3. Event Integrity

Outfield artifacts are derived from Infield events.

Event history is the single source of truth.

Public data must always be traceable to the originating event
within the MoCKA internal system.

## 4. Safety Rule

Sensitive operational data must never be published directly
from the Infield.

All Outfield artifacts must pass a sanitation or export process
before publication.

## 5. Role of the Outfield

The Outfield enables:

Public research dissemination  
AI collaboration interfaces  
Governance transparency  
Institutional memory accessible to external observers

The Outfield does not modify the internal state of MoCKA.

It reflects the system, but does not control it.
